# Marketing Markdown Sync Playbook

This project ships content-focused Markdown mirrors for key marketing routes in `public/*.md`.

The goal is to keep:

1. HTML marketing pages (`src/routes/*` + marketing components).
2. LLM discovery files (`public/llms.txt`, `public/llms-full.txt`).
3. Markdown mirrors (`public/*.md`).

in sync at every content change.

## Source of truth

1. Guide pages are sourced from `src/components/marketing/guides/guides-data.ts`.
2. Homepage and FAQ wording comes from:
   - `src/routes/index.tsx`
   - `src/components/marketing/sections/*`
3. About, pricing, and self-hosted pages come from:
   - `src/routes/about.tsx`
   - `src/routes/pricing.tsx`
   - `src/routes/self-hosted-3d-model-library.tsx`

## Required workflow after content changes

1. Update the route/component copy first.
2. Run:

```bash
bun generate:md
```

3. Review changed markdown files in `public/`.
4. Update `public/llms.txt` and `public/llms-full.txt` links when page targets change.
5. Run quality gate:

```bash
bun check
bun check-types
```

## Indexing policy

Markdown mirrors are intentionally excluded from SERP:

1. Server sets `X-Robots-Tag: noindex, nofollow, noarchive` for static `.md` responses.
2. Do not add `.md` URLs to `public/sitemap.xml`.
