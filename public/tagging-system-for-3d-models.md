# A tagging system for 3D models

Canonical: https://stl-shelf.com/tagging-system-for-3d-models

A practical tagging system for STL/3MF/OBJ libraries so you can find any model by project, part, or kit in seconds.
Folders alone cannot express all the relationships in a real 3D model library. Tags give you flexible structure without duplication.

## The problem

- A single model can belong to multiple projects, kits, or printer setups.
- Folder trees force one location and hide useful context.
- That leads to duplicate files and slow lookup during production work.

## Simple workflow

1. **Define a small tag taxonomy**: Start with project, part, kit, status, and material-focused tags.
2. **Tag every upload once**: Apply tags at upload time so search stays reliable as the library grows.
3. **Filter by combinations**: Use multiple tags together to find exact models without browsing folders.

## What STL Shelf is / is not

### STL Shelf is

- A private library for your STL, 3MF, and OBJ files
- Searchable tags that keep projects and parts organized
- Version history so every revision stays traceable
- Browser preview before you send models to print

### STL Shelf is not

- No marketplace
- No sharing or social feed
- No import or sync from other services

## FAQ

### How many tags should each model have?

Keep it practical: enough tags to find files quickly, but not so many that tagging slows you down.

### Can tags replace deep folder trees?

Yes. Tags reduce the need for rigid nested folders and duplicate storage.

### Do tags work across STL, 3MF, and OBJ?

Yes. The same tag system can organize mixed-format libraries.

### Does STL Shelf auto-import tags from other apps?

No. STL Shelf does not import or sync from external services.
